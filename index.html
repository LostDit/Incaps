Инкапсуляция в C++ private / public

Сокрытие данных и контролируемый доступ. Приватные члены класса недоступны извне.
Взаимодействие — только через публичный интерфейс (геттеры, сеттеры, методы).

Почему инкапсуляция важна

Скрытие данных

Приватные поля (private) защищены от случайного изменения. Внешний код не может испортить внутреннее состояние объекта.

Контроль доступа

Геттеры и сеттеры позволяют валидировать данные перед присваиванием. Например, баланс не может стать отрицательным без специальной логики.

Упрощение поддержки

Внутреннюю реализацию можно менять, не затрагивая код, который использует класс. Публичный интерфейс остаётся стабильным.

Повышение модульности

Класс становится самодостаточным компонентом. Легче тестировать, переиспользовать и понимать.

Пример класса BankAccount

class BankAccount {
private:
    double balance;   // инкапсулированное поле

public:
    double getBalance() const { return balance; }

    void deposit(double amount) {
        if (amount > 0) balance += amount;
    }

    bool withdraw(double amount) {
        if (amount > 0 && balance >= amount) {
            balance -= amount;
            return true;
        }
        return false;
    }
};

Прямое обращение к obj.balance вызовет ошибку компиляции — это и есть инкапсуляция.

Инкапсуляция

Механизм языка, объединяющий данные и методы в классе и ограничивающий доступ к деталям реализации. Отвечает на вопрос «как это сделано».

Абстракция

Концепция, скрывающая сложность и показывающая только необходимый интерфейс. Отвечает на вопрос «что делает объект».

Инкапсуляция — один из способов достижения абстракции. В C++ она реализуется с помощью спецификаторов private, protected, public.
